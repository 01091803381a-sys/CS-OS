# CS-OS
컴퓨터 구조 및 운영체제 수업 정리

## 📝 Class 1. 💻 CS-OS: 컴퓨터 구조 및 운영체제 요약
컴퓨터의 기본 작동 원리와 운영체제의 핵심 개념을 학습하고 정리하는 저장소입니다.

---

## 📖 1. 정보의 기초: 데이터와 명령어
컴퓨터가 처리하는 모든 정보는 크게 두 가지로 분류됩니다.

* **데이터(Data):** 컴퓨터가 이해하는 숫자, 문자, 이미지, 동영상과 같은 **정적인 정보**.
* **명령어(Instruction):** 데이터를 움직이고 컴퓨터를 실제로 **작동시키는 정보**.

---

## 🛠️ 2. 컴퓨터의 4가지 핵심 부품
컴퓨터를 구성하는 가장 핵심적인 하드웨어 요소입니다.

| 부품 | 역할 | 주요 구성 요소 / 특징 |
| :--- | :--- | :--- |
| **CPU** | 메모리의 명령어를 읽고, 해석하고, 실행함 | **ALU**(연산), **제어장치**(해석/제어), **레지스터**(저장) |
| **메모리(RAM)** | 현재 실행 중인 정보를 저장 | 주소(Address)를 사용해 접근, 휘발성 저장소 |
| **보조기억장치** | 보관할 정보를 장기 저장 | HDD, SSD 등 비휘발성 저장소 |
| **입출력장치** | 외부와의 데이터 교환 | 마우스, 키보드, 모니터 등 (I/O 장치) |

> **메인보드(Motherboard):** 위 부품들이 장착되는 판이며, 부품 간 데이터 통로인 **시스템 버스(System Bus)**가 존재합니다.
> * **주소 버스:** 주소 전달
> * **데이터 버스:** 명령어 및 데이터 전달
> * **제어 버스:** 제어 신호 전달

---

## 📊 3. 정보의 단위와 표현

### 🔹 Word (워드)
CPU가 한 번에 처리할 수 있는 데이터의 단위입니다.
* 종류: 하프 워드(Half), 풀 워드(Full), 더블 워드(Double)

### 🔹 2진법과 음수 표현 (2의 보수)
* **2진법(0b):** 0과 1로만 표현.
* **2의 보수:** 음수를 표현하는 방식. 2진수의 0과 1을 뒤집고 1을 더해 계산합니다. (상태 플래그가 1이면 음수)

### 🔹 16진법 (0x)
* 0~9와 A~F로 표현. 
* 2진법 4비트(2^4)가 16진수 1자리와 대응되어 변환이 쉬우므로 컴퓨터 분야에서 자주 사용됩니다.

---

## 🔠 4. 인코딩과 디코딩

* **인코딩(Encoding):** 문자를 0, 1로 이루어진 코드로 변환하는 과정.
* **디코딩(Decoding):** 코드를 다시 문자로 변환하여 해석하는 과정.

### 대표 문자 세트
1. **ASCII:** 7비트로 알파벳, 숫자 표현 (1비트는 Parity bit).
2. **EUC-KR:** 한글 완성형 인코딩 (2바이트).
3. **유니코드(Unicode):** 전 세계 문자를 통합한 집합. (UTF-8, UTF-16 등으로 구현)

---

## ⚙️ 5. 명령어와 주소 지정 방식

### 프로그래밍 언어의 단계
* **High-level:** 사람 중심 언어 (C, Python).
* **Low-level:** CPU 중심 언어 (기계어, 어셈블리어).
* **방식:** 컴파일(통째로 번역) vs 인터프리터(한 줄씩 번역).

### 명령어 구조와 오퍼랜드
* **명령어 = 연산 코드(Opcode) + 오퍼랜드(Operand)**
* **주소 지정 방식의 필요성:** 오퍼랜드 필드의 비트 수 제한 때문에, 더 넓은 메모리 영역을 가리키기 위해 **간접 지정** 등의 방식이 필수적입니다.
